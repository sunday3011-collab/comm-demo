# NR 大带宽子带拼接仿真

5G NR 100MHz / SCS=30kHz 的单 OFDM 符号仿真，对比两条解调路径：

- **路径 A（全带，黄金参考）**：4096 时域点直接做 4096-FFT，抽取 3276 个有效子载波。
- **路径 B（子带拼接，受芯片能力限制）**：把 100MHz 切成 5 个子带（RB=[55,54,55,54,55]），按**中射频(IF) + 基带(BB)** 两组件分工：
  - **中射频**：`DDC → 数字低通滤波（循环卷积）→ 下采样 4× → 施加实际传递时延`；
  - **基带**：`1024-FFT → 相位补偿（FIR 群延迟 + 补偿时延）→ 拼接成全带 3276`。

最终量化 A、B 两路的**最大/平均 幅度差异与相位差异**（补偿后为主指标，同时给出未补偿原始值）。

## 中射频→基带 传递时延建模

模拟"中射频把每个子带下采样数据传给基带"存在传递时延，以及基带补偿该时延。每个子带可独立配置两个属性（见 `ACTUAL_DELAYS` / `COMP_DELAYS`，单位=下采样域样点，可为小数）：

| 属性 | 作用环节 | 说明 |
|---|---|---|
| `actual_delay` | 中射频（FFT 前时域） | 实际传递时延，对下采样后的 1024 点做循环时延 |
| `comp_delay` | 基带（频域补偿） | 补偿时延，对各子载波 bin 施加 `exp(+j2π·m·comp_delay/1024)` |

- **`actual_delay == comp_delay`**：时延相位偏差被完全抵消，拼接结果与全带处理差异很小（与无时延时一致）。
- **二者失配**：补偿后相位差按 `360·m·Δdelay/1024` 度增大（边缘子载波 m≈330 时，0.5 样点失配 ≈ 58°）。

## 关键参数

| 参数 | 值 |
|---|---|
| ADC 采样率 Fs | 122.88 MHz |
| 子载波间隔 SCS | 30 kHz |
| 有效子载波数 | 3276（273 RB × 12） |
| 全带 IFFT/FFT | 4096 点（一个符号去 CP） |
| 子带数 / RB 分配 | 5 / [55,54,55,54,55] |
| 下采样因子 | 4 → 30.72 MHz |
| 子带 FFT | 1024 点（30.72M/1024 = 30kHz，每 bin 一个子载波） |

`SCS/Fs = 1/4096`、`30.72M/1024 = 30kHz`，使每个 FFT bin 恰好对齐一个子载波。子带中心取**整数子载波索引**，
保证 DDC 频率是 SCS 整数倍、搬移后子载波落在整数 bin 上，实现无 bin 间泄漏的干净拼接。

## 环境与运行

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python nr_subband_stitch.py
```

控制台打印四项指标（幅度/相位 × max/mean，含 raw 与补偿后），并生成 `result_qpsk.png`、`result_ones.png`
（幅度差、相位差、幅度谱叠加，竖线标子带边界）。对随机 QPSK 与全 1 平坦谱各跑一次。

## 参考结果

```
自检(全带参考 vs 输入)最大误差: ~1e-15        # 验证映射/IFFT/FFT 正确
补偿后(时延匹配):  幅度差 max≈1.3e-3 dB,  相位差 max≈2.3e-3 deg   # QPSK, 主指标
原始(未补偿):      幅度差 同上,           相位差 max≈180 deg       # FIR 群延迟+传递时延
全 1 平坦谱:       补偿后相位差 max≈1e-11 deg                     # 残差只剩滤波器幅度纹波
时延失配对照:      补偿后相位差 max≈58 deg                        # comp 偏离 actual 0.5 样点
```

## 误差来源

补偿掉已知的 FIR 群延迟后，残差集中在**子带边界子载波**，来自：
1. 低通滤波器的通带纹波与边缘滚降；
2. 邻带有限阻带衰减带来的泄漏（下采样后混叠）。

滤波器设计良好时（本例 Kaiser 窗、80dB 阻带、109 taps），两者均极小。把 `design_lpf()` 改差（缩短 taps、
降低阻带）即可放大误差，直观看到子带拼接对滤波器质量的敏感性。
