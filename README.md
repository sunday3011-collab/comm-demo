# NR 大带宽子带拼接仿真

5G NR 100MHz / SCS=30kHz 的单 OFDM 符号仿真，对比两条解调路径：

- **路径 A（全带，黄金参考）**：4096 时域点直接做 4096-FFT，抽取 3276 个有效子载波。
- **路径 B（子带拼接，受芯片能力限制）**：把 100MHz 切成 5 个子带（RB=[55,54,55,54,55]），逐子带做
  `DDC → 数字低通滤波（循环卷积）→ 下采样 4× → 1024-FFT → 取有效子载波`，再拼接成全带 3276 个频域数据。

最终量化 A、B 两路的**最大/平均 幅度差异与相位差异**（补偿子带 FIR 群延迟后为主指标，同时给出未补偿原始值）。

## 关键参数

| 参数 | 值 |
|---|---|
| ADC 采样率 Fs | 122.88 MHz |
| 子载波间隔 SCS | 30 kHz |
| 有效子载波数 | 3276（273 RB × 12） |
| 全带 IFFT/FFT | 4096 点（一个符号去 CP） |
| 子带数 / RB 分配 | 5 / [55,54,55,54,55] |
| 下采样因子 | 4 → 30.72 MHz |
| 子带 FFT | 1024 点（30.72M/1024 = 30kHz，每 bin 一个子载波） |

`SCS/Fs = 1/4096`、`30.72M/1024 = 30kHz`，使每个 FFT bin 恰好对齐一个子载波。子带中心取**整数子载波索引**，
保证 DDC 频率是 SCS 整数倍、搬移后子载波落在整数 bin 上，实现无 bin 间泄漏的干净拼接。

## 环境与运行

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python nr_subband_stitch.py
```

控制台打印四项指标（幅度/相位 × max/mean，含 raw 与补偿后），并生成 `result_qpsk.png`、`result_ones.png`
（幅度差、相位差、幅度谱叠加，竖线标子带边界）。对随机 QPSK 与全 1 平坦谱各跑一次。

## 参考结果

```
自检(全带参考 vs 输入)最大误差: ~1e-15        # 验证映射/IFFT/FFT 正确
群延迟补偿后:  幅度差 max≈1.3e-3 dB,  相位差 max≈2.3e-3 deg   # QPSK, 主指标
原始(未补偿):  幅度差 同上,           相位差 max≈180 deg       # 即 FIR 群延迟斜坡
全 1 平坦谱:   补偿后相位差 max≈1e-11 deg                     # 残差只剩滤波器幅度纹波
```

## 误差来源

补偿掉已知的 FIR 群延迟后，残差集中在**子带边界子载波**，来自：
1. 低通滤波器的通带纹波与边缘滚降；
2. 邻带有限阻带衰减带来的泄漏（下采样后混叠）。

滤波器设计良好时（本例 Kaiser 窗、80dB 阻带、109 taps），两者均极小。把 `design_lpf()` 改差（缩短 taps、
降低阻带）即可放大误差，直观看到子带拼接对滤波器质量的敏感性。
